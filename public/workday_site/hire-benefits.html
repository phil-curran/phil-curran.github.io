Process Overview

Overview:

This document was written for ISC Benefits Partners / Benefits Office Partners in order to assist in processing Special Handling situations within the Hire – Benefits (FSPD X.XX) and Enroll in Benefits (FSPD X.XX) processes.

While Workday is configured to handle most benefits enrollment and administration tasks automatically, or in response to employee actions, several employment scenarios require that a Benefits Partner evaluate the situation, make a decision, and complete a manual administrative override in Workday to modify an employee’s Benefit Eligibility.

Different classes of employees are, by default, eligible for different types of benefits. Within each of these classes, certain employment parameters (e.g., FTE percentage, length of appointment, prior benefits eligibility, etc.) can vary, and can qualify or disqualify an employee for benefits eligibility.

The following instructions outline which types of Special Handling conditions exist for each class of employee, and instruct the user on making the necessary modifications in Workday.

Initiation:

This process is initiated in response to several reports, which the Benefits Partner will receive via Workday Inbox messages. These reports indicate a number of special handling situations requiring manual intervention & processing, including:

  • Data returned from INT174 (prior UWRP Retirement Participation History)
  • Data returned from INT123 (prior DRS Retirement Participation History)
  • 1st Business Day
  • Entered after Hire Date
  • 6 months or less appointment
  • Return from Layoff
  • Rehire w/ less than 1 month calendar break
  • Interagency Transfer
  • Retiree Emeritus
  • Faculty returning within 12 months
  • Annual or Shorter appointment
  • Dual PEBB Enrollment

Different classes of employees are eligible for different types of benefits. Within each of these classes, certain employment parameters (e.g., FTE percentage, length of appointment, etc.) can vary, and can qualify or disqualify an employee for benefits eligibility.

Process Steps

The following walk-through demonstrates the Workday process of modifying one Employee's default benefit eligibility by creating Benefit Group / Eligibility Rules.

With an understanding of the rules which govern Benefit Eligibility, and an understanding of how to create Benefit Group / Eligibility Rules, you will be able to respond to Reports indicating Special Handling situations, evaluate each according to Eligibility Rules, and make corresponding corrections in Workday.

Example Scenario: 6 months or less

In the following walk-through, we will take action on a report, notifying that an employee named Boris Badenov, who is a fixed-term professioanl staff with an appointment of less than six months. We will use Workday to create special rules that confirm the employee's ineligibility for PEBB and Retirement benefits.

Security Group: Unit Initiator

Security Role(s):HCM Initiate 2, HR Partner

1. Locate the Employee:

Locate the Employee in question using the Global Search field:

2. View the Employee's current eligibility:

Click the Related Actions button (),

Select the Benefits Section, and

Select the 'View Benefits Eligibility' section.

Field Requirements Definitions:
Field: Definition:
Country The Country in which the employee/affiliate resides in.

By viewing the employee’s Benefits Eligibility, we can see that Boris is currently marked as eligible for the PEBB and Retirement plans available to Classified Staff.

However, given that you received the Special Handling report for Boris, you know that Boris’s appointment six months or less disqualifies him from eligibility for PEBB and Retirement options.

3. Create a Rule Override:

Navigate to the 'Change Benefits' section:

Click the Related Actions button (),

Select the Benefits Section, and

Select the 'Change Benefits' section.

Within the 'Additional Data' tab,

Find the “Benefit Group / Category” section

Click the 'Edit' button.

4. Specify Override Criteria:

Here, we will create rule overrides for PEBB and Retirement Eligibility.

Click on the “Add” button () to add a row to the table.

In the new row, complete the following fields:

Row Number: Start with 1 as the Row Number

Effective Date: Add the effective date of the change. Since the change is active at the start of the employee’s start date, the effective date should be the start date.

Benefit Group Category: Select “Not PEBB.” This specifies that the employee is not eligible for PEBB coverage.

Reason: Specify the reason why you are disqualifying the employee from this eligibility. Per the terms of Boris’ employment, which is less than 6 months, we would select the reason code “Less than or equal to 6 months.”.

Additionally, repeat steps 1-5 above to specify ineligibility for Retirement benefits.
Specify this as Row 2
Use the same Effective Date
Benefit Group / Category: select “Not Retirement”
Reason: select “Less than or equal to 6 months”

Click the “OK” button to finalize the changes.

5. Review Changes:

Next, review the changes you’ve made, confirm their accuracy, and click the “OK” button when you’re finished.

6. Confirm Changes:

Return to the Employee's main profile page by clicking on their name. We can now see the changes we’ve made to Boris’ Benefits Eligibility by viewing the “Benefits Group / Category” section.
